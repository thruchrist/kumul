KUMUL AI

Privacy & Technical Overview

"Wok mas go yet" — Work is out there. KUMUL is Papua New Guinea's dedicated WhatsApp Job Search Assistant, designed specifically for PNG job seekers with a heavy emphasis on user privacy and local context.

🛡️ Privacy First Architecture

We built KUMUL to be helpful, not intrusive. Users in PNG value discretion, so our system is explicitly designed to avoid the "Big Brother" feeling.

  • No Name Dropping: The AI never addresses the user by their name, even if provided. It feels like a neutral tool, not a CRM system.
  • No Database Mentions: The AI is strictly forbidden from using phrases like "I have saved this to your profile" or "In your records." It simply says "Got it!"
  • Context Only: Data is used purely to improve the immediate conversation (e.g., remembering you asked for mining jobs so it doesn't ask again). It is not used for targeted profiling.
  • No Data Sharing: Chat logs, extracted skills, or profiles are never sold, shared, or used for marketing.

⚙️ Technical Architecture

Designed to be resilient, fast, and optimized for low-bandwidth WhatsApp delivery.

  • Dynamic AI Fallbacks: Instead of crashing when an AI model hits a rate limit, the system automatically rotates through 8 different high-tier LLMs (Llama-3.3, GPT-OSS, Kimi-K2, Qwen, etc.) seamlessly in milliseconds.
  • Smart Message Chunking: Twilio limits messages to 1,600 characters. KUMUL automatically splits long responses (like job lists with URLs) into sequential, readable messages without breaking links.
  • Anti-Hallucination Tooling: When returning job results, the AI is physically bypassed from summarizing the output. The raw tool data (with clickable URLs) is sent directly to the user to prevent link-stripping.
  • PNG-Optimized Search Engine: Uses Tavily AI to scrape specifically curated PNG domains (PNGJobSeek, BSP Careers, Govt portals) rather than generic global searches, ensuring high relevance.

🗃️ Data Schema

The PostgreSQL database uses strict 3NF normalization.

  • User Profiles: Stores inferred data (target roles, location) using JSON fields for schema flexibility.
  • Interaction Logs: Captures intent, processing time, and tool usage for debugging, completely decoupled from user PII.
  • Saved Jobs: Uses strict unique constraints on URLs to prevent duplicate bookmarks for the same user.
